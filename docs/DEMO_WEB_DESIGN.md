# 浏览器白板 Demo 设计

日期：2026-07-16

## 目标

用最小实现验证产品闭环：用户在 PC 或 iPad 浏览器书写，提交后白板清空，服务端把页面截图交给视觉大模型，页面显示回答，并在本地保存截图、转录和回复。

## 已确认范围

- 浏览器前端，支持鼠标、触屏和 Apple Pencil。
- 手动“提交”与“清空”，暂不实现停笔自动提交。
- 不实现墨迹吸收、逐笔回复或淡出动画。
- Node.js 本地服务保存 API Key 并调用 OpenAI 兼容接口。
- 最近记忆作为文本上下文参与后续回答。
- 记忆使用本地 JSON 文件和 PNG 截图。
- 提供简单的历史记忆抽屉，不重放历史笔迹。

## 页面状态

```text
书写 → 提交中（白板立即清空）→ 显示回复 → 继续书写
  └──────────── 清空 ────────────┘
```

## 模块边界

- `public/app.js`：白板输入、截图导出、页面状态和接口请求。
- `server.js`：静态文件、配置、模型调用和记忆接口。
- `data/memories.json`：结构化记忆索引。
- `data/images/`：每轮用户白板截图。

## 安全边界

- API Key 只保存在服务端 `.env`。
- 服务默认监听局域网，以支持 iPad 访问。
- Demo 不含鉴权，只应运行在可信本地网络。
- 图片和近期文字记忆会发送到配置的大模型服务商。
