# 白板钢笔与纸张动画设计

日期：2026-07-19

## 目标

在现有浏览器白板、Supabase 记忆和智谱视觉调用链路上增加连续的纸张演出：钢笔质感书写、停笔自动提交、墨迹被纸张吸收、回答逐字写出，以及回答自动或手势清除。

## 已确认体验

- 用户停笔 3 秒后自动提交，不再使用“交给纸张”按钮。
- 笔画在提交后约 1.2 秒内以克制的纸张吸收效果淡去。
- 回答使用手写字体逐字显示。
- 回答完成后停留 12 秒，再用约 1.2 秒淡去。
- 回答停留期间，用户在纸张上滑动可提前触发淡出。
- 采用方案 A：真实感、清晰度与 iPad 性能平衡优先，不使用高成本粒子或 WebGL 特效。

## 状态机

```text
ready
  → writing
  → idle-countdown
  → absorbing
  → waiting
  → writing-answer
  → reading
  → fading-answer
  → ready
```

任何新笔画都会重置 3 秒倒计时。笔尖仍落下时不得提交。进入新状态时必须取消上一个状态创建的无效定时器，避免重复提交、回答交错或幽灵动画。

## 钢笔笔触

Canvas 继续接收 Pointer Events 和合并事件。每个采样点记录坐标、压力、时间和计算后的宽度。

- Apple Pencil 和支持压力的设备使用真实压力。
- 鼠标使用稳定的模拟压力。
- 笔宽由压力和移动速度共同决定：慢速笔画稍宽、快速笔画稍细。
- 使用中点二次曲线平滑相邻采样点，减少高速书写折线。
- 起笔和收笔使用轻微渐细，不追求精确书法笔锋。
- 墨色使用深褐黑和细微透明度变化，避免纯数字黑线的塑料感。

白板截图在任何吸收处理开始前生成，因此上传给视觉模型的始终是完整、清晰的原始墨迹。

## 自动提交与吸收

用户抬笔后启动 3 秒倒计时。倒计时期间再次落笔会取消旧倒计时。达到 3 秒且页面仍有墨迹时：

1. 立即导出 PNG。
2. 锁定新的书写输入。
3. 启动智谱请求。
4. 同时播放约 1.2 秒的墨迹吸收动画。

吸收动画使用 Canvas 快照叠层或画布透明度层实现：透明度逐渐降低，辅以轻微模糊和纸色覆盖。动画不修改已导出的 PNG，也不使用逐像素粒子模拟。

如果模型在吸收结束前返回，回答排队等待；如果吸收先结束，纸面进入低干扰等待状态。

## 回答书写

回答层使用项目内的 `DancingScript.ttf`。英文优先使用该字体，中文回退到系统楷体或手写字体。

- 普通字符间隔约 55ms。
- 空格时间更短。
- 逗号、分号和换行增加中等停顿。
- 句号、问号和感叹号增加更明显停顿。
- 回答区域使用正常换行，不允许长文本溢出纸张。

逐字完成后进入阅读状态并开始 12 秒计时。用户在纸面产生明确滑动距离时，提前进入淡出状态；单次点击不清除回答。淡出结束后恢复空白白板。

## 控件与反馈

- 移除“交给纸张”按钮。
- “清空”按钮在书写和倒计时期间可用，用于取消本轮内容。
- 状态文本依次表达“正在书写”“纸张正在聆听”“墨迹被纸张吸收”“纸张正在回应”等产品内语言。
- 记忆抽屉保持现有行为。

## 错误与无障碍

- 请求失败使用同一回答书写层逐字显示人格化错误，不弹出技术对话框。
- 页面隐藏、用户清空和状态重置时取消定时器与动画帧。
- `prefers-reduced-motion: reduce` 下跳过吸收渐变和逐字过程，直接进入对应最终状态，但保留自动提交与阅读时长。
- 动画层设置为不可交互，避免阻断触控与辅助技术。

## 测试与验收

- 单元测试覆盖停笔倒计时、计时器重置、标点延迟和状态转换。
- 浏览器测试覆盖鼠标、触摸和压力输入路径。
- 验证停笔 3 秒仅提交一次，且模型收到的 PNG 包含完整墨迹。
- 验证模型早返回和晚返回两种竞态顺序。
- 验证回答自动淡出和滑动提前淡出。
- 验证 PC 与 iPad 尺寸无横向溢出，动画期间控制台无错误。
- 验证 Cloudflare Workers 静态资源仍能被 Wrangler 识别。
